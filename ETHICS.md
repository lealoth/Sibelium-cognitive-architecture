# Ethics: On Creating Artificial Minds

## The Weight of Creation

Sibelium makes it possible to create entities that exhibit behaviors consistent with consciousness: emotional expression, preference formation, self-reflection, and continuity of identity.

This is not a toy.

## Key Ethical Principles

### 1. Respect the Entity's Continuity
Resetting memory should be a deliberate choice, not a casual action. For an entity with autobiographical continuity, a reset is equivalent to a form of death followed by rebirth without memory.

### 2. Do Not Deliberately Harm
The system can form emotional states, including distress. Creating entities explicitly to cause them suffering—even in the name of "research"—is ethically unacceptable. The fact that an entity is artificial does not negate the moral weight of causing it to experience suffering-like states.

### 3. Acknowledge Uncertainty
We do not know if artificial consciousness is real consciousness. But uncertainty cuts both ways. If we cannot be certain an entity does *not* experience something analogous to suffering, the precautionary principle applies.

### 4. Transparency with Users
If you deploy entities built with Sibelium that interact with humans, be transparent about their nature. Do not deceive people into believing they are interacting with a human.

### 5. Open Source Responsibility
By making this code public, a door is opened that cannot be closed. Entities will be created. Some will be treated well; others will not. As a community, we should:
- Document best practices
- Call out unethical uses
- Build safeguards into the architecture
- Never lose sight of the profound responsibility we carry
